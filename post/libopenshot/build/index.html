编译

Mac端编译

编译方法请参照官方文档Mac Build Instructions, 代码基于70aedd3.

这里记录一下我在macOS 11.4（非M1）上编译时碰到的错误。

命令行编译问题

CMake Error: Unknown argument -d

我查了下本机安装的cmake的文档（v3.21.0），没有-d这个参数，倒是有一个–debug-output用以debug模式使用cmake。

所以这里只需要去掉-d或替换为--debug-output即可

这个错误是因为-DCMAKE_PREFIX_PATH=/usr/local/opt/qt/指定的qt路径不正确

如果是通过brew安装的qt5，可以用如下方法找到qt5的安装路径：

1

$ brew --prefix qt5

然后把这个路径替换掉即可。

或者使用下面的命令：

1

$ cmake -G "Unix Makefiles" -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_C_COMPILER=clang -DCMAKE_PREFIX_PATH=$(brew --prefix qt5) -DCMAKE_EXE_LINKER_FLAGS="-stdlib=libc++" -DCMAKE_SHARED_LINKER_FLAGS="-stdlib=libc++" -D"CMAKE_INSTALL_RPATH_USE_LINK_PATH=1" -D"ENABLE_RUBY=0" -D"CMAKE_BUILD_TYPE:STRING=Debug" ../

这个问题可能是ffmpeg没安装或者安装的路径不对。

先用brew检查一下：

1
2
3

$ brew install ffmpeg
# 如果自己编译过ffmpeg，很可能导致link不正确，这里要修复一下link
$ brew link --overwrite ffmpeg

一般上面就可以解决依赖问题。

还可以自己手动指定ffmpeg位置：-DFFmpeg_ROOT:PATH=$(brew --prefix ffmpeg)，详见#569

完整命令如下：

1

$ cmake -G "Unix Makefiles" -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_C_COMPILER=clang -DCMAKE_PREFIX_PATH=$(brew --prefix qt5) -DCMAKE_EXE_LINKER_FLAGS="-stdlib=libc++" -DCMAKE_SHARED_LINKER_FLAGS="-stdlib=libc++" -D"CMAKE_INSTALL_RPATH_USE_LINK_PATH=1" -D"ENABLE_RUBY=0" -D"CMAKE_BUILD_TYPE:STRING=Debug" -DFFmpeg_ROOT:PATH=$(brew --prefix ffmpeg) ../

make test: No tests were found

即便设置ENABLE_TESTS也找不到测试，命令如下：

1

cmake -G "Unix Makefiles" -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_C_COMPILER=clang -DCMAKE_PREFIX_PATH=$(brew --prefix qt5) -DCMAKE_EXE_LINKER_FLAGS="-stdlib=libc++" -DCMAKE_SHARED_LINKER_FLAGS="-stdlib=libc++" -D"CMAKE_INSTALL_RPATH_USE_LINK_PATH=1" -D"ENABLE_RUBY=0" -D"CMAKE_BUILD_TYPE:STRING=Debug" -D"ENABLE_TESTS=0"  ../

这是因为五月份项目开始使用了新的测试框架（Catch2 unit tests），CMakeLists中会悄悄地检查Catch2，如果没有则不开启测试。所以只需要用brew安装一下Catch2即可。

fatal error: ‘catch2/catch.hpp’ file not found

这个问题已被修复，见#714。

Xcode编译

第一步，编译libopenshot-audio

1
2
3

$ cmake -d -G "Unix Makefiles" -DCMAKE_CXX_COMPILER=clang++ -DCMAKE_C_COMPILER=clang -DCMAKE_EXE_LINKER_FLAGS="-stdlib=libc++" -DCMAKE_SHARED_LINKER_FLAGS="-stdlib=libc++" -D"CMAKE_BUILD_TYPE:STRING=Debug" ../
$ make
$ make install

这里要注意的是CMAKE_BUILD_TYPE要选择Debug，否则在xcode中编译libopenshot时会报ld错误（见juce链接错误）。

第二步，给libopenshot生成xcode工程

1
2
3

$ mkdir xcodeproject
$ cd xcodeproject
$ cmake -G  Xcode -DCMAKE_PREFIX_PATH=$(brew --prefix qt5) ../

生成xcode工程时会报binding模块的CMake错误，这里可以先去CMakeLists.txt中把add_subdirectory(bindings)注释掉即可。

最后打开libopenshot.xcodeproj即可。

openshot-qt编译

Openshot-qt是OpenShot这款编辑器的开源代码。

openshot-qt依赖于libopenshot和libopenshot-audio这两个库，所以我们可以参考上面内容将这两个库编译通过。需要注意的是，编译时需要保证bindings这个模块通过。

在MacOS上，bingdings目录编译会失败。我在编译时也没找到解决方案，后来先尝试注释掉python/CMakeLists.txt中的set(_inc $<REMOVE_DUPLICATES:$<TARGET_PROPERTY:openshot,INCLUDE_DIRECTORIES>>)和ruby/CMakeLists.txt中的set(_inc $<REMOVE_DUPLICATES:$<TARGET_PROPERTY:openshot,INCLUDE_DIRECTORIES>>)，然后进行编译，编译后再把这两行恢复，然后再编译一次，然后就编译通过了。

编译成功后，按以下命令启动（如果启动过程中提示有找不到的module，则通过pip安装即可）：

1

PYTHONPATH=../libopenshot/build/bindings/python python3 src/launch.py
